## Objective

Build an OpenSearch-based SIEM platform and begin integrating pfSense logging.

## Accomplishments

### OpenSearch Deployment

- Installed OpenSearch
- Installed OpenSearch Dashboards
- Verified services are running
- Created initial SOC dashboard

### Logging Infrastructure

- Configured rsyslog to listen on UDP 514
- Verified rsyslog is receiving syslog traffic
- Confirmed UDP 514 listener is active

### pfSense Integration

- Configured remote syslog destination
- Verified network connectivity between pfSense and Ubuntu Agentic Server
- Tested connectivity with ICMP ping

### Troubleshooting

- Used tcpdump to inspect UDP 514 traffic
- Determined current syslog traffic appears to originate from Kali Linux
- Continued investigation into pfSense log forwarding

## Lessons Learned

- Successful network connectivity does not guarantee application-level logging.
- tcpdump is invaluable for validating packet flow.
- Building a SIEM requires validating each layer independently:
  - Network
  - Service
  - Log receiver
  - Indexing
  - Dashboard visualization
